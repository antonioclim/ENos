The hostname is the name of your Ubuntu system. At seminars, we will identify you by this name.

13. Common problems and solutions

‚ùå "WSL 2 requires an update to its kernel component" ‚ñº

Solution: Open PowerShell as Administrator and run:

üîµ POWERSHELL (Administrator) 
wsl --update

Then restart the computer.

‚ùå Virtualisation is not enabled (Error: 0x80370102) ‚ñº

Solution:

  1. Restart the computer
  2. When it starts, quickly press the BIOS key (usually Del, F2, F10, or F12 - depends on manufacturer)
  3. Look in menus for the option "Virtualization Technology", "VT-x", "AMD-V" or "SVM"
  4. Change from "Disabled" to "Enabled"
  5. Save and exit (usually the F10 key)
‚ùå Cannot paste commands in Ubuntu ‚ñº

Solution: In the Ubuntu window, use:

  • Right-click to paste, or
  • Ctrl+Shift+V (not just Ctrl+V)
‚ùå I forgot the password ‚ñº

Solution: Open PowerShell and run:

üîµ POWERSHELL 
wsl -u root

Now you are root (administrator). Change the password (replace yourname with your username):

‚¨õ BASH (as root) 
passwd yourname

Type the new password twice, then type exit to exit.

‚ùå SSH does not start ‚ñº

Solution: Run in Ubuntu:

‚¨õ BASH (Ubuntu) 
sudo apt install --reinstall openssh-server

Then start SSH:

‚¨õ BASH (Ubuntu) 
sudo service ssh start
‚ùå Cannot connect with PuTTY ‚ñº

Check:

  1. Is Ubuntu running? (the black window must be open)
  2. Is SSH started? Run in Ubuntu: sudo service ssh status
  3. Is the IP address correct? Run in Ubuntu: hostname -I
