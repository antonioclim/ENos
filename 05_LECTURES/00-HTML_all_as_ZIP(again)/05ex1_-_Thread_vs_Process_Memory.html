fork() createdes a COMPLETE COPY of the address space (Copy-on-Write for optimisation). Processes do not share memory directly - communication requires IPC.

P Process with Threads Same PID for all threads

Threads share code, data, heap and file descriptors. Each thread has only its own stack and registers. Creation: ~μs vs ~ms for processes.

Demonstration: Shared Variable Behaviour

Scenario 1: Threads

1. Global variable: counter = 0

2. Thread 1 executes counter++

3. Thread 2 sees the new value immediately!

Change visible instantly in all threads!

Scenario 2: Processes

1. Global variable: counter = 0

2. fork() → child modifies counter

3. Parent does NOT see the change!

Separate copies of the variable - complete isolation!
