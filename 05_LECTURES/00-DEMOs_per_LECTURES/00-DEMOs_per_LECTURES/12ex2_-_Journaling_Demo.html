how works Journaling-ul?

1. Write-Ahead Logging

Before modifying the disk, we write ALL intentions to the journal. The journal is persistent on disk.

2. Apply Changes

After the journal is written, we apply the changes to the main disk. Now we have redundancy.

3. Commit/Cleanup

Mark the transaction as complete. On recovery, we verify and re-apply incomplete transactions.
